Git routine.

 I did not know the git routine on linux so i chose the dark side. GitHub for Windows was of enormous help to get me started with GitHub and i was happily clicking away to commit and push for over a month. But the day had finally come for me to switch back to linux and learn how the git (command line) routine.

One starts off with generating an (unique) SSH key on your computer and adding this to your GitHub profile. Refer to this github help article to know how to generate said SSH keys and adding them.

NOTE: if the directory ~/.ssh does not exist, then create one. If you haven't used ssh on your system so far, there will not exist one.

Post authentication, sync your GitHub online repositories with your local ones, if you have such. One can use either $git pull https://github.com/reposity.git or
$git clone https://github.com/reposity.git to do so. The advantage of $git clone https://github.com/foobar.git over
$git pull https://github.com/foobar.git is that a remote origin will by default be added to which commits will be pushed. Simply put, the routine becomes $git pull https://github.com/foobar.git
$git remote add foobar https://github.com/foobar.git
$git push foobar or just $git clone https://github.com/foobar.git
$git push origin $git add . or
$git add folder/* can be used to check the current/specific directory for changes and staging them for commit. $git commit -m 'commit message' will commit the staged changes. One can also do $git commit -a -m 'commit message' to directly add changes, staging them and committing them at once. And finally,$git push origin/foobar will push the commits to the master branch (unless otherwise specified). $git status is a convenience command to check, at different points in time, what the changes are, what files are being staged for a commit or waiting to be pushed.
